API ENDPOINTS (EXAMPLES)

Authentication

  GET    /api/auth/profile/
  PUT    /api/auth/profile/update/

Products
  GET    /api/products/
  GET    /api/products/<slug>/
  GET    /api/products/<slug>/price-history/
  GET    /api/products/categories/

Cart
  GET    /api/orders/cart/
  POST   /api/orders/cart/add/
  PATCH  /api/orders/cart/items/<id>/
  DELETE /api/orders/cart/items/<id>/

Orders
  GET    /api/orders/
  GET    /api/orders/<id>/
  POST   /api/orders/checkout/

Payments
  POST   /api/payments/process/
  GET    /api/payments/methods/
  POST   /api/payments/<id>/refund/

DATABASE SCHEMA
═══════════════════════════════════════════════════════════════════════════════

Users Table
  - id (PK)
  - email (unique)
  - password (hashed)
  - first_name, last_name
  - phone_number
  - profile_picture
  - address fields
  - created_at, updated_at

Products Table
  - id (PK)
  - name, description
  - price, discount_price
  - category_id (FK)
  - stock, sku, brand
  - rating, num_ratings
  - created_at, updated_at

Categories Table
  - id (PK)
  - name, slug (unique)
  - description
  - parent_id (FK - self)
  - created_at, updated_at

PriceHistory Table
  - id (PK)
  - product_id (FK)
  - old_price, new_price
  - change_reason
  - created_at

Cart Table
  - id (PK)
  - user_id (FK, unique)
  - created_at, updated_at

CartItems Table
  - id (PK)
  - cart_id (FK)
  - product_id (FK)
  - quantity, price
  - created_at, updated_at

Orders Table
  - id (PK)
  - order_number (unique)
  - user_id (FK)
  - status, payment_status
  - total_amount, tax, shipping
  - shipping address fields
  - created_at, updated_at

OrderItems Table
  - id (PK)
  - order_id (FK)
  - product_id (FK)
  - quantity, unit_price, discount

Payments Table
  - id (PK)
  - order_id (FK)
  - payment_gateway, status
  - amount, transaction_id
  - created_at, updated_at
